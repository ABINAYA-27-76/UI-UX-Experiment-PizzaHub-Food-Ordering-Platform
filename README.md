# UI-UX-Experiment-PizzaHub-Food-Ordering-Platform

## Aim

To design and improve the user experience of a food ordering platform called PizzaHub by creating multiple interface screens, conducting A/B testing, and evaluating design variations to improve usability, engagement, and conversion rates.

## Objective
Design an attractive and user-friendly food ordering application.
Create multiple UI screens including Home, Product Listing, and Payment Page.
Develop two design variations (Variant A and Variant B).
Compare user interaction and usability through A/B testing.
Analyze which design performs better in terms of user engagement and order completion.

## Software and Design Tools Required
1.Figma
2.Canva
3.GitHub
4.Food/Pizza Images

## Project Description

PizzaHub is an online pizza ordering platform designed to provide users with a seamless food ordering experience. The project focuses on creating visually appealing interfaces, improving navigation, simplifying the ordering process, and optimizing payment workflows.

The project is divided into two phases:

### Phase 1 – Initial Design Development
1.Landing/Home Page Design
2.Pizza Menu/Product Listing Page
3.Secure Payment Page
### Phase 2 – A/B Testing and Design Enhancement
1.Variant A – Existing Design
2.Variant B – Improved Design
3.Performance Comparison
4.User Experience Evaluation

## Procedure
### Phase 1: UI Design Creation
Step 1: Home Page Design
* Created an attractive hero section.
* Added navigation menu.
* Added promotional banners and offers.
* Included customer statistics and social proof.
* Added call-to-action buttons.
Step 2: Product Listing Page
* Created category filters.
* Added pizza cards with images.
* Included ratings and delivery information.
* Added discount badges.
* Added "Add to Cart" functionality.
Step 3: Payment Page
* Designed secure checkout layout.
* Added payment methods.
* Displayed order summary.
* Included address information.
* Added coupon and discount section.

### Phase 2: A/B Testing
1.Variant A
 * Traditional layout.
 * Standard product display.
 * Basic checkout interface.
 * Minimal promotional elements.
2.Variant B
 * Modern visual hierarchy.
 * Enhanced call-to-action buttons.
 * Improved product visibility.
 * Better trust indicators.
 * Stronger promotional banners.
 * Conversion-focused layout.
## Workflow

## A/B Testing Analysis
1.Variant A
 * Simple interface.
 * Easy navigation.
 * Traditional layout structure.
2.Variant B
 * Better visual engagement.
 * Improved user attention.
 * Enhanced trust and credibility.
 * Higher conversion potential.
 * Findings

Variant B showed better user engagement due to:

 * Strong visual hierarchy.
 * Better CTA placement.
 * Improved promotional visibility.
 * Enhanced trust indicators.
 * Modern UI design.

## Result

Thus, a complete PizzaHub food ordering platform was successfully designed using modern UI/UX principles. Multiple screens were created and enhanced through A/B testing. The comparison between Variant A and Variant B demonstrated that improved visual hierarchy, promotional placement, and user-focused design significantly enhance user engagement and conversion rates. Therefore, Variant B was identified as the most effective design solution.
